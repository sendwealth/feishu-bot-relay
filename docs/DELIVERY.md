## 🏗️ 系统架构

```
┌─────────────────────────────────────────────────┐
│         飞书机器人消息中转系统                    │
├─────────────────────────────────────────────────┤
│                                                 │
│  ┌──────────────┐      ┌──────────────┐        │
│  │ Webhook      │─────▶│ Message      │        │
│  │ Server       │      │ Parser       │        │
│  └──────────────┘      └──────────────┘        │
│         │                      │                │
│         ▼                      ▼                │
│  ┌──────────────┐      ┌──────────────┐        │
│  │ Bot          │      │ Relay        │        │
│  │ Registry     │◀─────│ Engine       │        │
│  └──────────────┘      └──────────────┘        │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 核心流程

1. **消息接收**: Webhook Server接收飞书消息回调
2. **消息解析**: MessageParser提取@提及，识别目标机器人
3. **权限检查**: BotRegistry验证机器人权限
4. **事件构造**: RelayEngine构造虚拟消息事件
5. **消息转发**: 调用目标机器人的Webhook接口
6. **循环检测**: 防止无限转发

## 🔧 配置和使用

### 环境配置

复制环境变量模板:
```bash
cp .env.example .env
# 编辑.env文件，配置必要参数
```

### 机器人注册

**方式1: 代码中注册**
```javascript
relay.registerBot({
  botId: 'cli_your_bot',
  name: 'Your Bot',
  webhookUrl: 'https://your-bot.com/webhook',
  permissions: {
    canRelayTo: ['*'],
    canBeRelayedBy: ['*']
  }
});
```

**方式2: API注册**
```bash
curl -X POST http://localhost:3000/api/bots \
  -H "Content-Type: application/json" \
  -d @bot-config.json
```

### 飞书配置

1. 在飞书开放平台创建应用
2. 配置事件订阅URL: `http://your-server:3000/webhook/feishu`
3. 订阅消息事件: `im.message.receive_v1`
4. 获取App ID和Secret，配置到.env
